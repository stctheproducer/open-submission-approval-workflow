# Delta - assessor-handoff

## ADDED

### Rubric-friendly submission overview

- Given an assessor opens the root README, when they scan the document, then they can find the submission framing, system shape, setup, workflow, testing, deployment, and trade-off guidance in a review-friendly order.
- Given the assessor wants to understand the whole submission quickly, when they read the README, then they can tell how to run, evaluate, and reason about the project without reconstructing missing context from the codebase.

### Verified submission facts

- Given the submission is finalized, when the README describes setup, workflow, testing, and deployment, then those statements match the implemented and verified system rather than a planned one.
- Given a detail is not yet verified, when the README is finalized, then it is called out plainly instead of being implied as complete.
