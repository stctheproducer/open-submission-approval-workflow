# Delta - operator-workflow

## ADDED

### Canonical local setup path

- Given a fresh checkout, when a maintainer follows the repository guidance, then they can install dependencies, prepare local configuration, start local services, apply database changes, seed data, and start development through one repeatable path.
- Given a documented step no longer works, when the maintainer reaches it, then the guidance surfaces the gap instead of implying the project is ready.

### Local verification before handoff

- Given the local environment is running, when a maintainer checks the operator instructions, then they can confirm the app is usable with the documented commands.
- Given the operator flow changes, when the guidance is updated, then the repository keeps one honest local verification path rather than several competing ones.
