# Delta — application-draft-reopening

## ADDED

### Applicant can reopen a changes-requested application back to draft

- Given an application has been returned for changes and the acting user is the owning applicant, when the applicant chooses to reopen it, then the application returns to draft so it can be edited and resubmitted.
- Given an application has not been returned for changes, when the applicant attempts to reopen it, then the request is rejected and the application remains unchanged.
- Given a user is not the owning applicant, when they attempt to reopen a changes-requested application, then the request is rejected and the application remains unchanged.
